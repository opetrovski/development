Catalog Manager Integration Package

Welcome to the integration package of Catalog Manager. This package helps you integrate your applications or other external systems with Catalog Manager. It provides the public Web services and application programming interfaces (APIs) of Catalog Manager as well as additional resources, templates, samples, utilities, and documentation.

Package Content

This package contains the following items:

  • Platform services. Catalog Manager exposes its basic functionality as public Web services, the so-called platform services. Applications and external systems that are integrated with Catalog Manager invoke the functionality of the platform services. The services are accessed by SOAP messages over HTTP/HTTPS.
    • oscm-extsvc-platform.jar - a library containing the API of the platform services
    • API javadoc - the javadoc for the API of the platform services
    • WSDL files - the WSDL files for the platform services with their related schema files
  • Resources that support you in implementing a provisioning service for integrating an application with the subscription management of Catalog Manager:
    • oscm-extsvc-provisioning.jar - a library with the API for implementing a provisioning service
    • API javadoc - the javadoc for the provisioning service API
    • WSDL file - the WSDL file a provisioning service must conform to with its related schema file
  • Resources that support you in adapting the login/logout implementation of a Java-based Web application you want to integrate with Catalog Manager:
    • oscm-webservices-proxy.jar - a library containing a Web service proxy for BASIC, CLIENTCERT, and STS Web service ports of the Catalog Manager platform services. This proxy is implemented in Java and ready to be used. It creates a Web service proxy containing all information required to establish a connection with a Catalog Manager Web service.
    • Integrationhelper.jar - a library containing a token handler and logout listener which are implemented in Java and ready to be used.
    • oscm-api-client-handler.jar - A library containing a SOAP handler for adding a service version number to the outbound SOAP messages of a Catalog Manager Web service client. The service version can be found in the tag included in the WSDL files.
    • Integrationhelper.war - a Web archive including the integration helpers and additional resources required by them.
    • readme.txt - Readme file containing information on how to use the integration helpers and the sample implementations.
    • Integration helper sources - a ZIP file containing the source code of the integration helpers.
    • Licenses - third-party licenses required for the libraries which are included in the Web archive.
  • Resources that support you in implementing technical service operations for an application which are to be accessible from the Catalog Manager user interface:
    • oscm-extsvc-operation.jar - a library with the API for implementing a Web service that provides the technical service operations
    • API javadoc - the javadoc for the operation service API
    • WSDL file - the WSDL file an operation service must conform to with its related schema file
  • Resources that support you in integrating Catalog Manager with a process control system:

    For implementations based on SOAP/WSDL:

    • oscm-extsvc-notification.jar - a library with the API for implementing a Web service that notifies the process control system of actions carried out in Catalog Manager
    • API javadoc - the javadoc for the notification service API
    • WSDL file - the WSDL file a notification service must conform to with its related schema file

    For implementations based on REST:

    • API specification - YAML files describing the REST API
  • Resources that support you in integrating a parameter configuration tool:
    • configurationApi.js - a Java script library for integrating a parameter configuration tool with Catalog Manager
    • oscm-integration-parameter-configurator.zip - a sample JavaScript tool which can be integrated with Catalog Manager. Here is the description of this sample.
  • Resources that support you in integrating Catalog Manager with an external billing system:
    • oscm-billing-external-interfaces.jar - a library with the billing plug-in API for implementing a billing adapter which forms the connection between Catalog Manager and the external billing system
    • API javadoc - the javadoc of the billing plug-in API
    • Sample - a sample billing adapter that works with a simple file-based billing system which is also included. Here is the description of this sample.
  • Documentation:
    • Overview - a PDF manual introducing Catalog Manager
    • Developer's Guide - describing the public Web services and application programming interfaces of Catalog Manager and how to integrate applications and external systems with Catalog Manager
    • Technology Provider's Guide - a PDF manual for technology providers describing how to prepare applications for usage in a SaaS model and how to integrate them with Catalog Manager
    • Marketplace Owner's Guide - a PDF manual for marketplace owners describing how to manage and customize marketplaces in Catalog Manager
    • Supplier's Guide - a PDF manual for suppliers describing how to define and manage service offerings for applications that have been integrated with Catalog Manager
    • Reseller's Guide - a PDF manual for resellers describing how to prepare, offer, and sell services defined by suppliers on a marketplace
    • Broker's Guide - a PDF manual for brokers describing how to suppurt suppliers in establishing relationships to customers by offering their services on a marketplace
  • XML schema files:
    • TechnicalServices.xsd - the schema for a technical service definition
    • MarketableServices.xsd - the schema for a marketable service definition
    • BillingResult.xsd - the schema for XML files resulting from an export of billing data
    • Locale.xsd - a schema included in the BillingResult.xsd
    • SupplierRevenueShareResult.xsd - the schema for XML files resulting from an export of supplier revenue share data
    • ResellerRevenueShareResult.xsd - the schema for XML files resulting from an export of reseller revenue share data
    • BrokerRevenueShareResult.xsd - the schema for XML files resulting from an export of broker revenue share data
    • MPOwnerRevenueShareResult.xsd - the schema for XML files resulting from an export of marketplace owner revenue share data
    • BillingBase.xsd - a schema included in the SupplierRevenueShareResult.xsd, ResellerRevenueShareResult.xsd, BrokerRevenueShareResult.xsd, and MPOwnerRevenueShareResult.xsd
  • Catalog Manager version information

    • Best Practices for Implementing SOAP-based Web Services

    Consider the following when implementing Web services based on SOAP and WSDL for integrating applications or external systems with Catalog Manager:

    • Web services based on Java:

      It is recommended to use the JAX-WS framework for implementing a Web service. The interfaces for all Web services are shipped in .jar files in this package. For calls from Catalog Manager to your Web service (outbound calls), you can directly write your Java code and include the appropriate Catalog Manager .jar file. For calls from your Web service to Catalog Manager (inbound calls), you implement the relevant Catalog Manager interfaces. No code generation is required.

    • Web services NOT based on Java:

      In this case, the Web services MUST conform to the WSDL and XSD files provided in this package. Depending on the platform and the Web services framework you are using, code generation is required or dynamic usage is possible.

    Refer to the Developer's Guide for further information.
